## 仕様書の標準構成

各機能仕様書は以下の構成に従って作成します。

```markdown
# [機能名] 統合設計書（[ユースケースID範囲]）

## 1. 概要
### 1.1 目的
### 1.2 対象ユースケース（表形式）
### 1.3 追加コンポーネント数

## 2. データベース設計
### テーブル定義（カラム、インデックス）

## 3. APIエンドポイント設計
### エンドポイント一覧
### リクエスト/レスポンス定義

## 4. Pydanticスキーマ設計

## 5. サービス層設計

## 6. フロントエンド設計
### 6.1 画面一覧
### 6.2 コンポーネント構成

## 7. 画面項目・APIマッピング
### 7.1 一覧画面
| 画面項目 | 表示形式 | APIエンドポイント | レスポンスフィールド | 変換処理 |

### 7.2 作成/編集画面
| 画面項目 | 入力形式 | 必須 | APIエンドポイント | リクエストフィールド | バリデーション |

## 8. ユースケースカバレッジ表
| UC ID | 機能名 | API | 画面 | ステータス |
```
